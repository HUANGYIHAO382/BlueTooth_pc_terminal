# HeartRateMonitor 项目结构与模块解耦说明

本文档用于帮助你把这个项目的某些功能「拿出来」单独用在其他项目里。


## 2. 可解耦的核心模块

### 2.1 `Blegetheartbeat.py` — BLE 心率采集

**依赖**：`bleak`（pip install bleak）、`asyncio`（标准库）

**协议说明**：使用 **BLE 标准心率服务**
| 名称 | UUID |
|------|------|
| 心率服务 | `0000180d-0000-1000-8000-00805f9b34fb` |
| 心率测量特征 | `00002a37-0000-1000-8000-00805f9b34fb` |

> **重要**：这套 UUID 是蓝牙官方标准（SIG 定义），只有支持该标准的设备（手环、胸带等）才能用这套代码读取心率。
> 如果你的设备用的是厂商私有协议（常见 UUID：`FFF0/FFF1`、`FFE0/FFE1` 等），需要另外分析特征和数据格式。

> **与本仓库 PC 客户端（`pc_ble_client`）配合时**：界面里「瑞光血压计」与「心率手环」二选一，不要用手环的 MAC 却选血压角色。心率服务 `0x180D` 与瑞光透传 `0xFFF0` 在 128 位 UUID 里共用 SIG 基底，旧版若用「只比 UUID 尾部」会误把心率服务当成 FFF0，随后订阅 FFF1 会失败；已在本仓库用规范展开比较修复。

**如何在其他项目里使用**：

```python
import asyncio
from Blegetheartbeat import BLEHeartRateMonitor

monitor = BLEHeartRateMonitor()

# 1. 设置回调：收到心率时会调用此函数
def on_heart_rate(timestamp, bpm):
    print(f"[{timestamp}] 心率: {bpm} BPM")

monitor.heart_rate_callback = on_heart_rate

async def main():
    # 2. 扫描设备
    devices = await monitor.scan_devices()
    for d in devices:
        print(d.name, d.address)

    # 3. 用地址连接（address 来自扫描结果）
    success, msg = await monitor.connect_device("XX:XX:XX:XX:XX:XX")
    if success:
        await asyncio.sleep(30)  # 保持 30 秒
    await monitor.disconnect_device()

asyncio.run(main())
```

**返回的心率数据格式**：
- `timestamp`：字符串，`YYYY-MM-DD HH:MM:SS`
- `bpm`：整数，次/分钟

### 2.2 `udp_pusher.py` — UDP 心率推送（需要 PyQt5）

**依赖**：`PyQt5`（用 QTimer 做定时）、`socket`（标准库）

**如果你的其他项目没有 PyQt5**，可以把定时逻辑改成 `threading.Timer` 或 `asyncio` 定时协程，核心发送函数只有几行：

```python
import socket, time

def send_hr_udp(bpm: int, host="255.255.255.255", port=18500):
    """向局域网广播一条心率 UDP 报文"""
    msg = f"HR,{bpm},{int(time.time()*1000)}".encode("utf-8")
    sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    sock.setsockopt(socket.SOL_SOCKET, socket.SO_BROADCAST, 1)
    sock.sendto(msg, (host, port))
    sock.close()
```

**报文格式（UTF-8 明文）**：
```
HR,<bpm>,<epoch_ms>
示例：HR,72,1713248123456
```

## 3. 事件循环说明（asyncio + PyQt5 共存）

这个项目用了 `qasync` 把 PyQt5 的事件循环和 asyncio 合并：

```python
from qasync import QEventLoop
app = QApplication(sys.argv)
loop = QEventLoop(app)
asyncio.set_event_loop(loop)
with loop:
    loop.run_forever()
```

**如果你的其他项目是纯 Python（没有 Qt）**：直接用 `asyncio.run()` 或 `asyncio.get_event_loop().run_until_complete()` 即可，`Blegetheartbeat.py` 本身不依赖 Qt。

## 4. 关于 BLE 设备兼容性

### 标准协议（本项目支持）
- UUID：`180D` + `2A37`
- 常见设备：Polar H10 胸带、部分运动手环（需在广告中声明 Heart Rate Service）

### 私有协议（本项目不支持，需单独适配）

| 常见 UUID 组合 | 典型品牌/用途 |
|---------------|-------------|
| `FFF0 / FFF1` | 多种国产廉价手环、血压计 |
| `FFE0 / FFE1` | HM-10 模块、部分健康手环 |
| `6E400001-...` | Nordic UART Service（NUS） |

**判断方法**：用 Bleak 枚举设备上所有服务和特征：

```python
async def print_all_services(address: str):
    from bleak import BleakClient
    async with BleakClient(address) as client:
        for service in client.services:
            print(f"[服务] {service.uuid}  {service.description}")
            for char in service.characteristics:
                print(f"    [特征] {char.uuid}  {char.properties}")
```

把输出结果中的 UUID 贴到文档或反馈给我，可以帮你判断需要订阅哪个特征。

## 5. 最小可运行示例（不依赖 Qt，纯 asyncio）

把 `Blegetheartbeat.py` 和 `system_utils.py` 复制到你的项目，即可用下面的代码单独跑：

```python
import asyncio, logging
from Blegetheartbeat import BLEHeartRateMonitor

logging.basicConfig(level=logging.INFO)

monitor = BLEHeartRateMonitor()

def on_hr(timestamp, bpm):
    print(f"{timestamp}  →  {bpm} BPM")

monitor.heart_rate_callback = on_hr

async def main():
    print("正在扫描...")
    devices = await monitor.scan_devices(timeout=5.0)
    for d in devices:
        print(f"  {d.name}  {d.address}")

    addr = input("输入要连接的设备地址: ").strip()
    ok, msg = await monitor.connect_device(addr)
    print(msg)
    if ok:
        try:
            await asyncio.sleep(60)  # 采集 60 秒
        except KeyboardInterrupt:
            pass
        await monitor.disconnect_device()

asyncio.run(main())
```

## 6. 你遇到的 BLE 连接错误分析

日志显示：

```
已检测到 FFF0，开始订阅 FFF1…
FFF1 未就绪 BleakCharacteristicNotFoundError …（重试 5 次）
连接失败: Characteristic 0000fff1-... was not found!
```

**根本原因**：你的其他项目尝试订阅 `FFF0/FFF1`（私有协议），但设备上**不存在** `FFF1` 这个特征，或者特征完整 UUID 与代码里写的不匹配。

**排查步骤**：

1. 用上面「枚举所有服务」的代码，把这个设备的完整服务/特征树打印出来。
2. 找到真正携带心率数据的特征 UUID（通常在 properties 里有 `notify`）。
3. 把找到的 UUID 替换掉代码里的 `FFF1`。

**也有可能**：这台设备支持标准 `180D/2A37`，你的其他项目选错了协议。可以先用本项目的 `Blegetheartbeat.py` 尝试连接，看是否能正常获取心率。
